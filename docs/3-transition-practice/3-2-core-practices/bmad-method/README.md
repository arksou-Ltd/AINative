# BMAD-METHOD 详解

## 系统化的 AI 协作软件开发方法论

BMAD-METHOD (Building Modern Applications with AI-Driven Development) 是一套经过实践验证的AI协作软件开发方法论，帮助Builder实现高质量、高效率的软件交付。

## 什么是 BMAD-METHOD？

BMAD-METHOD 是一个完整的AI协作软件开发框架，它将软件开发的全流程系统化，并通过专业AI智能体和引导式工作流，帮助Builder实现端到端的高质量交付。

### 核心组成

```
BMAD-METHOD
├── 4个开发阶段（完整覆盖软件生命周期）
├── 3条专业轨道（方法论 + 构建工具 + 创意系统）
├── 21个专业AI智能体（分工明确的AI协作伙伴）
└── 50+引导式工作流（标准化的执行步骤）
```

### 核心价值

1. **系统化**：不是零散的AI工具使用，而是完整的方法论体系
2. **专业化**：21个专业AI智能体，各司其职
3. **标准化**：50+引导式工作流，确保每个环节有章可循
4. **高质量**：通过专业分工和质量检查，达到90-100分的商业标准

## 4个开发阶段

### 阶段1：头脑风暴与产品定义
**目标**：从想法到清晰的产品定义

- 创意生成和评估
- 用户研究和需求分析
- 市场研究和竞品分析
- 产品愿景和PRD编写

**关键输出**：Product Brief、PRD文档

---

### 阶段2：架构设计与技术选型
**目标**：技术可行性和架构设计

- 技术栈选型
- 架构模式设计
- 技术债务评估
- API和数据模型设计

**关键输出**：Architecture Document、Tech Spec

---

### 阶段3：Epic分解与Story拆分
**目标**：将PRD拆解为可执行的开发任务

- Epic定义和优先级
- Story拆分和验收标准
- Sprint规划
- Story Context生成

**关键输出**：Epic文件、Story文件、Sprint Status

---

### 阶段4：迭代开发与质量保障
**目标**：高质量代码交付

- AI结对编程
- 代码审查
- 测试和质量保障
- 持续集成和部署

**关键输出**：可发布的软件产品

## 3条专业轨道

### BMad Method (BMM)
**软件开发方法论轨道**

专注于软件开发全流程的系统化方法论，包含：
- 产品定义工作流
- 架构设计工作流
- 开发实施工作流
- 质量保障工作流

### BMad Builder (BMB)
**智能体和工作流构建轨道**

专注于构建和管理AI智能体和工作流，包含：
- 创建新的AI智能体
- 设计引导式工作流
- 模块化管理
- 文档自动生成

### Creative Intelligence System (CIS)
**创意智能系统轨道**

专注于创新和创意生成，包含：
- 创意头脑风暴
- 设计思维工作流
- 问题解决方法论
- 故事叙述框架

## 为什么选择 BMAD-METHOD？

### 传统方式 vs BMAD-METHOD

| 维度 | 传统碎片化AI使用 | BMAD-METHOD |
|------|------------------|-------------|
| **系统性** | 零散工具使用 | 完整方法论体系 |
| **专业性** | 通用AI助手 | 21个专业AI智能体 |
| **标准化** | 无标准流程 | 50+引导式工作流 |
| **质量** | 不稳定 | 90-100分商业标准 |
| **学习曲线** | 需要大量摸索 | 有章可循，快速上手 |

## 快速开始

1. **安装BMAD-METHOD**
   ```bash
   # 克隆仓库
   git clone https://github.com/yourusername/bmad-method
   cd bmad-method

   # 安装依赖
   npm install
   ```

2. **初始化项目**
   ```bash
   # 运行项目初始化工作流
   /bmad:bmm:workflows:workflow-init
   ```

3. **开始第一个产品**
   ```bash
   # 运行产品定义工作流
   /bmad:bmm:workflows:product-brief
   ```

4. **学习和实践**
   - 阅读[如何使用](how-to-use.html)指南
   - 查看[核心特点](core-features.html)了解详细架构
   - 理解[质量保障机制](quality-guarantee.html)
