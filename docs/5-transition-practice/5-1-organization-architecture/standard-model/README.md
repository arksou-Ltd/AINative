# 标准型 AI Native 组织

## 完全AI Native的组织架构设计

标准型AI Native组织是完全基于AI Native理念设计的组织架构，适用于新产品开发、新团队组建和创业公司。

### 章节内容

- [核心原则](core-principles.html) - AI Native组织的基本原则
- [团队规模](team-scale.html) - 完全灵活的团队规模设计
- [角色定义](role-definition.html) - Builder核心角色及其他角色
- [工作方式](work-style.html) - 端到端、PDCA的工作模式

### 核心特征

| 维度 | 传统组织 | 标准型AI Native组织 |
|------|----------|---------------------|
| **团队规模** | 固定（按职能划分） | 完全灵活（按需调整） |
| **核心角色** | 专业分工 | Builder（端到端业务能力） |
| **工作方式** | 流程驱动 | 结果驱动（PDCA） |
| **协作模式** | 人与人 | 人与AI，端到端负责 |
| **质量保障** | 流程审核 | 专业团队优化 |

### 关键优势

1. **极致灵活**：团队规模可从1人到N人动态调整
2. **高效产出**：Builder+AI可达到传统团队60-80分的质量
3. **成本优化**：减少协调成本，提高人效
4. **快速迭代**：端到端负责，决策链路短

### 适用场景

✅ **推荐使用：**
- 新产品从0到1开发
- 创业公司快速验证
- 需要快速迭代的项目
- 小规模高效团队

❌ **暂不推荐：**
- 大规模现有产品（建议用过渡型）
- 强合规要求的传统行业
- 需要大量跨部门协作的场景
