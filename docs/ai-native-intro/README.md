# AI Native 简述：章节导航

本章节集群用于快速让读者形成“认知-类比-方法-时机”闭环，并可作为 PPT 演示与文档模式的统一素材。内容围绕组织能力壁垒、第一性原理、核心类比、60-80-100 模型以及“为什么是现在”五个主题展开，覆盖理解深度、实践落地、传播触达三类成功标准。[Source: docs/PRD.md#Executive-Summary][Source: docs/PRD.md#Success-Criteria]

## 背景

- AI Native 关注“组织能力壁垒”而非单点工具使用，核心是让小团队在高自动化环境下持续复用、持续发布。[Source: docs/PRD.md#Project-Classification]
- 目标读者：管理者（决策与资源配置）、内容创作者（叙事与沉淀）、Builder（端到端交付）、转型负责人（节奏与标准）。
- 阅读顺序建议：`core-hook` → `first-principles` → `core-analogy` → `60-80-100-model` → `why-now`，既能满足快速讲解，也能支持深度研读。

## 概念

- **AI Native**：小而精的 Builder + AI + 专业团队协作，按结果拆分任务，依赖公共知识库与自动化校验，强调速度、复用与质量并存。[Source: docs/PRD.md#Product-Classification]
- **双模式交付**：文档模式用于深度阅读与沉淀，PPT 模式用于传播和决策沟通，二者共享同一内容骨架与图示素材。[Source: docs/architecture.md#Project-Structure]
- **成功标准映射**：理解深度（读者能准确复述壁垒与模型）、实践落地（能据此拆分任务与节奏）、传播触达（PPT/摘要便于分享）。[Source: docs/PRD.md#Success-Criteria]

## 解释

1) 为什么要用章节导航  
   - 减少信息搜寻成本：按主题分文件，便于在文档模式查阅、在 PPT 模式复用。  
   - 对齐期望：所有后续内容（案例、转化指南、PPT）以此骨架对齐，避免叙事漂移。  
   - 支撑成功标准：每个章节都需显式回应“理解-实践-传播”三类指标。

2) 如何使用本导航  
   - 快速演示：选择核心 Hook + 类比 + Why Now 三段即可在 5-8 分钟内完成价值阐述。  
   - 深度研读：按“第一性原理 → 模型 → 场景与时间窗口”顺序阅读，结合案例章节补充。  
   - 复用素材：图示占位与引用占位集中列出，方便设计/制作与核验数据。

## 意义

- 帮助团队在任意沟通场景下用统一话术解释 AI Native：是什么、为什么、怎么做、为什么是现在。
- 为后续案例与转化路线提供共用框架，减少重写成本，提升传播一致性。
- 让管理者、Builder、内容创作者在同一“坐标系”下讨论优先级与风险。

## 实践指引（用于落地与对齐）

- 会议/演示用法：  
  - 5 分钟版本：讲 Hook + 类比 + Why Now；用一张对比图 + 一张时间线即可。  
  - 15 分钟版本：加入第一性原理与 60-80-100 模型，展示“怎么做”与“如何迭代”。
- 编写/扩写流程：  
  - 先套用“背景→概念→解释→意义”骨架，填充引用与图示占位，再做 AI 校对与一致性检查。  
  - 每次增补数据/案例后，将可复用片段抽到公共引用区，减少后续重复。
- 质量检查清单：  
  - 是否引用成功标准（理解/实践/传播）并给出例证？  
  - 是否有引用/数据占位与图示占位？路径与命名是否遵循 kebab-case？  
  - 是否明确读者可以“怎么用”内容（演示/学习/决策）？

## 成功标准映射示例

- 理解：读者能准确复述 Hook、三大原理、模型与时机，并区分“工具使用”与“组织重构”。  
- 实践：能照骨架拆分任务，生成可用初稿，再按 60-80-100 节奏迭代；能调用公共引用/图示。  
- 传播：能在 PPT/邮件/会议中用统一话术讲清楚，减少上下游认知偏差。
