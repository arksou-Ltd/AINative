Product Brief: AINative

Date: 2025-12-15 Author: Jett Context: 知识分享与实践指导平台

Executive Summary

AINative是一个介绍文档网页项目，旨在帮助个人和组织理解AI Native理念，并提供从传统组织向AI Native组织转型的实践指南。项目将通过GitHub Pages部署，提供PPT演示模式和详细文档模式两种交互方式，兼顾概念传播与实操落地。

Core Vision

Problem Statement

核心问题1：理解鸿沟

  • 许多人对"AI Native"的理解停留在表面（"使用AI工具"），而非组织能力的根本性重构
  • 缺乏清晰、系统的概念框架帮助人们理解AI Native的本质
  • 现有资料碎片化，难以形成完整认知

核心问题2：实践困境

  • 理解了概念，但不知道如何开始转型
  • 缺乏可落地的实践路径和方法论
  • 没有针对不同组织现状的差异化转型方案（标准型 vs 过渡型）

核心问题3：交流传播障碍

  • 需要向团队、合作伙伴讲解AI Native理念时，缺乏有效的演示工具
  • 深度阅读与快速传播的需求难以兼顾

Proposed Solution

AINative介绍文档网站 - 一个双模式的知识与实践平台

解决方案特点：

  1. 系统化内容体系

    • AI Native简述：从第一性原理出发，清晰定义核心概念
    • 实际案例：真实的AI Native组织案例与数据验证
    • 转化路程：标准型组织架构 + 过渡性方案，因地制宜

  2. 双模式交互设计

    • PPT演示模式：全屏演示，适合演讲、团队分享、快速传播
    • Document文档模式：左侧目录导航 + 详细内容，适合深度学习、查阅参考

  3. 实践工具栈整合

    • 详细的BMAD-METHOD实践框架
    • 推荐工具栈（Cursor + Claude Code + Linear）
    • Builder能力培养路径

  4. 社区互动机制

    • 联系方式（微信 + 邮箱）促进交流与反馈

Key Differentiators

1. 双模式设计，兼顾传播与学习

  • PPT演示模式：打破传统文档的局限，支持现场演讲与团队分享
  • Document模式：系统化深度内容，支持自学与查阅

2. 全用户覆盖，不分职级与背景

  • 从CEO到普通员工，从技术到非技术，AI Native是所有人的基础能力
  • 内容分层：概念理解 → 看到结果 → 实操落地 → 结合实际

3. 实践导向，工具栈明确

  • 非技术人员：Cursor + AI（门槛最低）
  • 技术人员：Claude Code CLI / Gemini CLI / Codex（进阶选择）
  • 管理者：Linear + Slack + Github + Devin（团队协作）
  • 实际上只需要Cursor就可以开始

4. 因地制宜的转型方案

  • 标准型AI Native组织（理想状态）
  • 过渡型方案（基于现有组织结构渐进式转型）
  • 双轨制策略（新产品vs现有产品）

Target Users

核心理念：AI Native是所有人的基础能力

不分职级、不分技术背景，AINative网站面向所有想理解和实践AI Native的人。

决策层用户

角色： CEO、CTO、团队负责人

核心关注点：

  • ❓ 转型后是什么样？ 标准型AI Native组织的形态与特征
  • ❓ 组织如何重构？ 扁平化、按结果分工、Builder角色定义
  • ❓ 需要什么样的人才？ Builder能力模型，5维能力框架
  • ❓ 如何最大化AI赋能？ 工具栈选择、流程设计、团队协作

使用场景：

  • PPT演示模式：向高层、团队演示AI Native愿景，获得支持与共识
  • Document深度阅读：制定转型战略、设计组织架构、规划人才培养

工具栈需求：

  • Linear（项目管理）
  • Slack（团队协作）
  • Github（代码管理）
  • Devin（AI自主开发）

期望成果： 制定出结合自身企业情况的AI Native转型方案。

执行层用户 - 技术人员

角色： 开发者、工程师、技术负责人

核心关注点：

  • ❓ 如何落地实践？ BMAD-METHOD工作流、开发者工具栈
  • ❓ 用什么工具？ Cursor、Claude Code CLI、Gemini CLI、Codex
  • ❓ 如何提升能力？ Builder技能路径、从60分到80分

使用场景：

  • Document深度学习：系统学习BMAD-METHOD、工具配置、最佳实践
  • 实操练习：跟随指南配置工具、运行示例项目

工具栈需求（按门槛排序）：

  1. Cursor（必备，门槛最低） ⭐
  2. Claude Code CLI（进阶）
  3. Gemini CLI（进阶）
  4. Codex（进阶）

期望成果： 能独立使用AI工具完成开发任务，成为Builder。

执行层用户 - 非技术人员

角色： 产品、运营、设计、市场等非技术岗位员工

核心关注点：

  • ❓ 我也需要AI？ 理解AI Native不只是技术人员的事
  • ❓ 我能做什么？ 在自己的工作领域如何应用AI
  • ❓ 门槛高吗？ 非技术人员的AI工具入门

使用场景：

  • PPT演示模式：快速理解AI Native基本理念
  • Document选择性阅读：重点关注"转化路程"中的非技术实践

工具栈需求：

  • Cursor + AI（足够满足基本需求） ⭐
  • 学会"AI协作"而非"编程"

期望成果： 能在日常工作中有效使用AI提升效率，不再畏惧技术工具。

创业者 / 个人项目

角色： 独立开发者、创业者、solopreneur

核心关注点：

  • ❓ 小团队怎么做？ 1-3人的AI Native实践
  • ❓ 从0到1怎么做？ 新产品开发的完整流程
  • ❓ 成本多少？ 工具栈的经济性（Cursor $20/月为主）

使用场景：

  • 全流程实践：从概念到落地，完整走一遍
  • 快速验证：用最小成本验证AI Native模式

工具栈需求：

  • Cursor（核心，$20/月）
  • Claude Code CLI（可选）
  • Linear（免费版足够）
  • Github（免费）

期望成果： 用Builder + AI模式独立完成产品开发。

MVP Scope

Core Features（缺一不可）

1. 内容体系（完整3章）

1.1 AI Native 简述

  • 核心Hook："组织能力才是公司真正的壁垒"
  • 第一性原理：能力边界重新定义、角色根本转变、组织架构革命
  • 核心类比："传统组织 vs 拥有全球顶尖科学家的组织"
  • 完整定义：Builder + AI + 专业团队，60-80-100分模型
  • 为什么是现在：大模型的理解能力与自主程度突破

1.2 AI Native 案例

  • 成功公司案例

    • OpenAI ($13B ARR, $1B月收入)
    • Cursor AI ($2.6B估值, <100人)
    • Commonwealth Bank (84%用户不愿回退)
    • Bupa APAC (410,000+行AI辅助代码)

  • 组织转型案例

    • Amazon (15% IC/Manager比例提升)
    • Moderna (合并技术与HR部门)
    • Bayer (裁减50%管理层)
    • NVIDIA (Jensen Huang 40+直接下属)

  • 市场数据验证

    • AI Native应用开发工具市场: $26.99B (2025)
    • 企业AI采用率: 79%组织已采用AI agents
    • 开发者AI工具使用: 50%每天使用，90%将在2028年采用

1.3 转化路程

组织架构转型：

  • 标准型AI Native组织

    • 核心角色：Builder（综合性人才）
    • 工作方式：端到端负责，PDCA循环
    • 协调机制：通过算法而非层级

  • 过渡性组织架构

    • 双轨制策略：新产品全AI Native，现有产品渐进式
    • 微观层面：单个版本/需求使用AI Native
    • 宏观层面：保持现有流程，最大化AI使用

实践工具栈：

  • 基础层（所有人）

    • Cursor - 日常编码 ($20/月)

  • 进阶层（技术人员）

    • Claude Code CLI - 架构设计、深度推理
    • Gemini CLI / Codex - 备选方案

  • 管理层（团队负责人）

    • Linear - 项目管理
    • Slack - 团队协作
    • Github - 代码管理
    • Devin - AI自主开发 ($500/月)

  • 框架层（完整项目）

    • BMAD-METHOD - AI辅助开发工作流
    • 21个专业智能体，50+工作流
    • v6架构，90%成本节约

Builder能力培养路径：

  • Level 1: AI素养（1-2月）- 理解AI、基础提示工程
  • Level 2: 工具精通（2-3月）- Cursor熟练使用
  • Level 3: 架构能力（3-6月）- 独立完成60-80分产品
  • Level 4: 专业优化（6-12月）- 建立专业团队网络
  • Level 5: 组织变革（12+月）- 推动团队转型

  • 影响： 高跳出率，低实践转化
  • 缓解措施：
    • 核心Hook设计强有力（"组织能力是壁垒"）
    • 类比法降低认知成本（"顶尖科学家团队"）
    • PPT模式提供快速理解路径
    • 逐步引导式内容结构

风险2：工具栈实践门槛

  • 描述： 非技术人员可能对Cursor等工具畏惧
  • 影响： 理解与实践脱节
  • 缓解措施：
    • 强调"只需要Cursor"即可开始
    • 提供详细的Cursor配置指南
    • 案例演示降低心理门槛
    • 视频教程（未来可考虑）

风险3：内容更新维护

  • 描述： AI领域变化快，内容可能快速过时
  • 影响： 数据失效，案例老旧
  • 缓解措施：
    • 聚焦核心理念（第一性原理不易变）
    • 工具栈标注时间（2025年推荐）
    • GitHub开源，社区可贡献更新
    • 预留更新机制

风险4：传播受限

  • 描述： 依赖自然传播，缺乏推广资源
  • 影响： 触达用户有限
  • 缓解措施：
    • 内容质量优先（深度理解者自然传播）
    • PPT模式便于分享演示
    • 联系方式促进交流与口碑传播
    • GitHub + 社交媒体自然流量

Technical Preferences

推荐技术方案

方案A：VuePress（推荐）

  • 优点： Vue生态成熟，插件丰富，支持Markdown原生
  • PPT模式： 自定义Vue组件 + reveal.js集成
  • 构建： Vite（快速）
  • 适合： 内容为主，需要灵活定制

方案B：Docusaurus（备选）

  • 优点： React生态，Facebook出品，文档体验好
  • PPT模式： MDX + reveal.js
  • 构建： Webpack
  • 适合： 重文档体验，不需要太多定制

方案C：自定义实现（最灵活）

  • 技术栈： Vite + Vue 3 / React + Tailwind CSS
  • PPT模式： reveal.js / impress.js / swiper
  • 构建： 完全自主控制
  • 适合： 需要高度定制化交互

推荐选择：方案A（VuePress）或 方案C（自定义）

  • 如果追求快速上线 → VuePress
  • 如果追求完美体验 → 自定义实现
